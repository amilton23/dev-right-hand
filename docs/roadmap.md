# Roadmap

## Phase 1

- Structure contracts, agents, and local tracking.
- Create a CLI to scan directories.
- Cover the main flow with unit tests.

## Phase 2

- Read real test and coverage artifacts.
- Add Git integration.
- Enrich rules with parsing for `pyproject.toml`, `requirements.txt`, and pipelines.

## Phase 3

- Support parallel execution.
- Persist tracking in a configurable backend.
- Add export to JSON, Markdown, and PR comments.

## Phase 4

- Integrate with distributed observability.
- Add drift evaluation and continuous monitoring.
- Support agents with tool use and asynchronous workflows.
